# Классификация рамановских спектров мозговой ткани: ансамблевый подход с использованием стекинг-мета-модели

**Краткое описание.** Представлено решение задачи автоматической классификации образцов биологической ткани по спектрам комбинационного рассеяния света с целью определения экспериментальной группы: контроль, эндогенная или экзогенная экспрессия белка HSP70. Использованы предобработка спектров (коррекция фона, сглаживание Савицкого–Голея, L2-нормировка), ансамбль из четырёх базовых классификаторов (LightGBM, CatBoost, Ridge, SGD) и стекинг с логистической регрессией в качестве мета-модели. Оценка проводилась при строгой Group K-Fold валидации по файлам источников. На тестовой выборке (n = 25 200) итоговая модель (Stacking) достигла accuracy 60,73% и F1 macro 0,60. Лучше всего распознаётся класс экзогенной экспрессии (recall 0,82, F1 0,67); для endo характерен высокий precision (0,86) при умеренном recall (0,48); контроль даёт precision 0,48 и recall 0,56. Реализация доступна в виде воспроизводимого пайплайна и веб-сервиса для предсказания по одному спектру.

---

## 1. Введение

Рамановская спектроскопия позволяет получать спектральные отпечатки биологических образцов без разрушающей пробоподготовки и используется в том числе для дифференциации типов тканей и состояний, связанных с экспрессией определённых белков. В рамках поставленной задачи необходимо по спектру комбинационного рассеяния отнести образец мозговой ткани к одному из трёх классов: **control** (контроль), **endo** (эндогенная экспрессия HSP70), **exo** (экзогенная экспрессия HSP70). Дополнительно учитываются регион мозга (кора, стриатум, мозжечок и др.) и параметр центра съёмки (1500 или 2900), что отражает вариативность экспериментальных условий.

Спектры представлены в виде зависимостей интенсивности от волнового числа в области примерно 930–2000 см⁻¹. Задача является типичной многоклассовой классификацией при ограниченном объёме данных и выраженной межиндивидуальной вариабельности, поэтому целесообразно применение ансамблевых методов и строгой схемы валидации, исключающей утечку информации между обучением и тестом.

Цель работы — построить воспроизводимый пайплайн предобработки спектров и классификации с использованием ансамбля моделей и стекинг-мета-модели, оценить метрики на независимой тестовой выборке и обеспечить возможность применения модели в виде веб-сервиса.

---

## 2. Материалы и методы

### 2.1. Данные

Исходные данные — спектры рамановской спектроскопии в табличном виде (столбцы: волновое число, интенсивность). Сбор в единый датасет выполняется скриптом `build_dataset.py` из сырых файлов, организованных по папкам, соответствующим классам (control, endo, exo). Для каждого спектра фиксируются метаданные: регион мозга, параметр центра, идентификатор файла-источника. Идентификатор источника используется при разбиении на обучающую и тестовую выборки с целью не допустить попадания спектров одного и того же образца одновременно в train и test (Group K-Fold).

### 2.2. Предобработка спектров

Предобработка выполнялась в соответствии с рекомендованной методикой и включала три последовательных шага.

1. **Коррекция фона:** для каждого спектра из значений интенсивности вычитался минимум по этому спектру, что устраняет постоянное смещение фона.
2. **Подавление шума:** сглаживание фильтром Савицкого–Голея (окно 11, порядок полинома 3) для уменьшения высокочастотного шума при сохранении формы пиков.
3. **Нормировка:** L2-нормировка — каждый спектр приводится к вектору единичной длины, что снижает влияние различий в абсолютной интенсивности между измерениями.

### 2.3. Модели машинного обучения и ансамбль

В качестве базовых классификаторов использовались четыре модели различной природы.

- **LightGBM** и **CatBoost** — градиентный бустинг на деревьях решений; способны учитывать нелинейные зависимости и взаимодействия признаков (волновых чисел).
- **RidgeClassifier** и **SGDClassifier** — линейные модели; задают базовый уровень и повышают разнообразие ансамбля.

Итоговое предсказание строилось методом **стекинга**. По каждой базовой модели получались out-of-fold предсказания (вероятности по классам) на тех же разбиениях, что и при валидации. Эти вероятности объединялись в признаковое пространство для мета-уровня. **Мета-модель** — логистическая регрессия, обученная по объединённым вероятностям базовых моделей; на выходе — итоговый класс и вероятности по трём классам.

### 2.4. Схема валидации

Применялась кросс-валидация **Group K-Fold** по полю `source_file`: все спектры, принадлежащие одному файлу (образцу), оказывались либо только в обучающей, либо только в тестовой части разбиения. Это исключает оптимистичную оценку качества из-за корреляции между спектрами одного образца и приближает оценку к сценарию предсказания для новых, ранее не виденных образцов.

В финале от команды ожидалось предоставить следующее. Ниже указано, где это реализовано в данном решении.

## 4. Обсуждение и выводы

Применение стекинга поверх разнородных базовых моделей (два бустинга и две линейные модели) позволило улучшить общую точность и F1 по сравнению с лучшей отдельной моделью (CatBoost). Строгая Group K-Fold валидация по файлам-источникам снижает риск завышения метрик из-за зависимости между спектрами одного образца и даёт более реалистичную оценку для переноса на новые образцы.

Отрицательные значения R² отражают то, что задача решалась как классификация; интерпретация R² здесь вспомогательная. Ограничением остаётся умеренная абсолютная точность (60,73% при трёх классах и случайном угадывании 33,33%), что может быть связано с высокой межиндивидуальной и межэкспериментальной вариабельностью спектров, ограниченным объёмом обучающей выборки и спектральным перекрытием между классами. Дальнейшее улучшение может потребовать расширения набора данных, подбора признаков или областей спектра, а также учёта метаданных (регион, центр) в явном виде в модели.

В итоге реализован полный пайплайн: сбор датасета (`build_dataset.py`), предобработка, обучение ансамбля и стекинг-мета-модели, оценка и экспорт моделей в папку `model_export/` (нативные форматы для LGBM/CatBoost, joblib для Ridge, SGD и мета-модели). Развёрнут веб-сервис (`web/` на Flask), позволяющий загрузить файл спектра (колонки Wave и Intensity), указать регион мозга и центр и получить предсказание класса и вероятностей по трём классам. Таким образом, решение доведено до формы, пригодной для воспроизведения и практического использования.

## 5. Программное обеспечение и воспроизведение

- **Сбор данных:** Python 3, скрипт `build_dataset.py` (при необходимости — с аргументом лимита файлов на класс).
- **Обучение и оценка:** Jupyter-ноутбук `sample.ipynb` — загрузка данных, предобработка, разбиение по группам, обучение ансамбля, расчёт метрик, экспорт моделей в `model_export/`.
- **Зависимости:** pandas, numpy, scikit-learn, LightGBM, CatBoost, scipy (фильтр Савицкого–Голея).
- **Веб-сервис:** каталог `web/`, Flask; установка: `pip install -r requirements.txt`, запуск: `python app.py`. Интерфейс: http://127.0.0.1:5000 — загрузка файла с колонками Wave и Intensity, выбор региона и центра, получение класса (control / endo / exo) и вероятностей.
